# Project Rules

## Secrets Policy

Secrets (passwords, API keys, private keys, PSKs, setup keys,
tokens) MUST only exist in files that are universally and
unmistakably recognized as secret storage by humans, AI agents,
CI systems, and automated tools. These are:

- `.env` files (`.env`, `.env.*`)
- Files with `secret` or `credential` in the name
- Files inside directories named `secrets/` or `credentials/`
- Files with restrictive permissions (0600) AND documented as
  secret in the project

Secrets MUST NEVER appear in:
- Docker Compose files (`docker-compose*.yml`)
- Generated YAML/JSON config files
- Shell scripts
- Source code
- Log output
- README or documentation

When a generated file needs to pass credentials to a container,
use `env_file:` directives pointing to `.env.*` files — never
inline `environment:` values containing secrets.

## Private Data Leak Prevention

A pre-commit hook (`.githooks/pre-commit`) blocks commits
containing private data patterns. Patterns are stored in
`.githooks/blocked-patterns` (gitignored — never enters the
repo). One regex per line, lines starting with `#` are ignored.

To add a new pattern:
```bash
echo 'my-private-domain\.com' >> .githooks/blocked-patterns
```

The hook is activated by: `git config core.hooksPath .githooks`
(already configured for this repo).

## Secrets Policy

Files containing secrets in this project:
- `.env` — Mullvad, Tailscale, gluetun API credentials
- `secrets/vpn-instances.yaml` — VPN instance config with
  credentials (source of truth, local-only, never synced)
- `.env.vpn-*` — per-instance credentials generated by
  `generate-vpn.py` (synced to VPS, 0600 permissions)

The `secrets/` directory is gitignored (except `*.example`
files). All other files must reference secrets via variable
interpolation (`${VAR}`) or `env_file:` directives.
